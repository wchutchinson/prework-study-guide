Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The title element is important for SEO.
  • List items are nested in ordered or unordered blocks.
  • Tags represent the letters between < >.
  • Element is the content between the opening tag and closing tag.
  • <alt> are used to describe images and their inclusion can help webpage rankings for SEO.

CSS

  • A margin indicates how much space we want arouidn the outside of an element.
  • A padding indiciates how much space we want aroudn teh content inside an element.
  • Inline CSS applies to a specific HTML tag, is less fledible, and has to be edited everywhere it is applied.
  • Internal CSS is a style sheet that is imbedded in the header of the HTML using a <style> element
  • External CSS is in a seperate file in the project and is <link> in the header of the HTML page

Git

  • Git is a version control system.
  • GitHub stores code in the cloud.
  • Git Flow:
    1. Create a branch.
    2. Make changes
    3. Create a pull request
    4. Address and review comments
    5. Merge changes
    6. Delete Branch
  • Git Cheat Sheet>
  • Git Command Steps
    1. git checkout main - switches to main branch
    2. git pull origin main - pulls down latest code
    3. git checkout -b feautre/add-xxx - creates new branch
    4. git status - check what branch I am in
    5. git add -A - after code is changed this stages the changes
    6. git commit -m "change message"
    7. git pull origin main - syncs my feature branch with main branch
    8. git push origin feature/add-xxx - pushes changs to GitHub
